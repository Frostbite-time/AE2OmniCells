# AE2 Omni Cells

![banner](https://i.imgur.com/4Qds3if.png)

**AE2 Omni Cells** 提供一套统一的 AE2 存储方案：**在同一个存储元件中，同时存放多种不同类型的资源**。这里的“资源类型”指 AE2 生态中的**资源键（AE Key）**——例如**物品**与**流体**（以及由其他模组暴露为 AE Key 的内容）。容量覆盖 **1k → 256m**，并通过“**可同时存放的不同种类数量上限**”构成三条产品线（含便携版）。本模组**不**对“增加一种新种类”额外收取容量，只以“种类上限”进行行为约束。

---

## 这到底意味着什么（为什么叫 “Omni”）

* 你可以把**物品**与**流体**（以及其他被 AE2 识别的资源键）**放在同一张磁盘里**。
  例如：把“圆石、铁锭、红石、**液态经验**、**乳胶**、**粉红史莱姆**”等**同时**放进同一张 64k 磁盘，而不是分别用“物品磁盘”和“流体磁盘”。
* **不收“种类费”**：容量（bytes）只代表**总量**，与“当前有多少种不同东西”**无关**；只有当你尝试再加入一个**全新**种类且该盘的“种类上限”已满时，才会被拒绝。
* **对未提供 AE2 存储磁盘的附属也有效**：
  即使某些附属没有做“专用的 AE2 磁盘”，它们的资源（只要 AE2 可识别）也能**直接存入 Omni 元件**。

> 简单理解：**一个元件装“什么都行”**（Omni），但**能同时装多少“不同种”**由所选系列与容量决定；**总量**由容量决定。

---

## 产品线与上限策略

| 系列       | 同时存放的“不同种类数”上限             | 适合的阶段          |
| -------- | -------------------------- |----------------|
| **全能**   | 固定 **63** 种                | 游戏早期，资源紧张      |
| **复杂全能** | 随容量提升（例：1k≈12 → 256m≈6400） | 游戏发展到中期，有足够的资源 |
| **量子**   | **无限**                     | 游戏后期，有大量资源可用   |

> 提示：三系列的**容量梯度**一致（1k / 4k / 16k / 64k / 256k / 1m / 4m / 16m / 64m / 256m），每一档都有**便携版本**，便携与磁盘在容量/上限上保持一致。

---

## 与传统 AE2 存储的区别

* 过去常见做法是把“物品、流体”等分开管理；**Omni** 的思路是**在同一个元件里同时放不同类型**。
* 你**不会**因为“增加了一个新种类”而损失更多容量；容量只代表总量。
* 行为边界是**“种类上限”**——达到上限后，再加入“全新种类”会被拒绝，但**已有种类**仍可继续增量存入（只要容量还够）。

---

## 数据与安全

* **小 NBT 策略**：物品本体只保存必要的预览信息（用量、状态等），**真正的存储数据写在独立容器**中，降低大 NBT 风险。
* **存档位置**：数据位于世界存档的 `data/ae_universal_cell_data`。
* **容错读取**：无法读到的条目不会立即清除；进入世界时会再次尝试，以适应模组环境变化。

## 常见问题

**Q：我能把 **灵魂涌动** 的监守者灵魂存进来吗？**

A：比较麻烦，但是可以。虽然**Soulplied Energistics**提供了资源键识别，但是却没有提供输入总线和输出总线的完整适配。你只能使用元件IO端口与存储总线配合来存入。我推荐你额外安装**Applied Soul**来补全输入总线和输出总线的逻辑。

**Q：便携元件与普通元件的区别？**

A：容量与“种类上限”一致，仅形态与使用场景不同。
